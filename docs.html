Getting started

Rhino sets up entirely inside Telegram. You never need a dashboard open for the agent to keep working.

  1. Open @rhino_agent on Telegram and start a chat.
  2. Pick a persona (or write your own).
  3. Fund the agent’s dedicated card.
  4. Set spend caps and approval thresholds.
  5. Step back — Rhino trades and reports in the same chat.
First trade tip

Start with a small daily limit and an approval threshold. Raise them once you’ve watched a few cycles.

Personas

A persona is the agent’s thesis, temperament, and risk preference. Rhino uses it on every sizing and rebalance decision.

Presets

  • The Analyst — measured entries, low turnover, weeks-long horizon.
  • The Degen — high velocity, tight stops, liquidity-aware sizing.
  • The Guardian — drawdown-first, concentration checks, months-long horizon.

Custom

Write a short thesis, name it, and attach risk rules. Rhino runs that configuration exactly — nothing else leaks into its decision loop.

// lock a preset
/persona set analyst

// or define your own
/persona create "night-ops"
/persona thesis "rotate into liquid large-caps on vol spikes"

Funding & cards

Every Rhino agent gets its own funded card tied to a dedicated account. The agent cannot see or spend from your other balances.

  • Fund with USDC (or supported stables on Robinhood Chain).
  • The card number is masked in chat (••••0402 style).
  • Withdrawals require an explicit Telegram command from you.
/fund 2500 usdc
→ card active · balance synced

Guardrails

Autonomy sits inside fences you define. Rhino will not request overrides for hard limits.

  • Daily spend cap — absolute max the agent can commit per day.
  • Per-trade limit — hard ceiling on a single order.
  • Approval threshold — trades above this pause for /approve or /skip.
  • Kill switch — /kill halts all trading immediately.
/limit 300/day
/limit trade 150
/approve threshold 100
/kill on

Telegram commands

/statusOpen positions, card balance, guardrail hits
/persona …View, set, or create a persona
/fund …Add capital to the agent card
/limit …Set daily and per-trade caps
/approveAllow a paused order above threshold
/skipDismiss a paused order
/killEmergency halt — trading stops now
/digestForce a P&L and activity summary

x402 payments

x402 is the HTTP-native payment standard Rhino uses for data and execution costs. When a priced endpoint returns 402 Payment Required, the agent pays in stablecoin and retries — no API keys sitting in config.

  1. Agent calls a priced resource.
  2. Server responds 402 with payment requirements.
  3. Rhino attaches an X-PAYMENT header (e.g. EIP-3009 USDC).
  4. Request settles on-chain and returns 200 — usually in a couple of seconds.
  5. The settlement line is logged to your Telegram audit channel.
// simplified handshake
POST /chain/execute
status: 402 Payment Required
X-PAYMENT: "eip3009:0x9f…c402"
status: 200 OK — order filled

Robinhood Chain

Trades settle on Robinhood Chain so positions, cash, and history remain yours — not trapped inside an opaque broker wallet. Rhino talks to Trading and Banking surfaces over MCP, then posts fills and payments back into Telegram as they land.

Security model

  • Dedicated card and account — no access to the rest of your portfolio.
  • Hard spend limits the agent cannot raise.
  • One-tap kill switch in Telegram.
  • Every trade and payment writes an immutable audit line you can read in chat.

FAQ

Can Rhino spend more than I funded?

No. The agent only spends from its dedicated card balance and never past the daily / per-trade caps you set.

What happens when a trade is above my approval threshold?

Rhino posts a preview to Telegram and waits. Reply /approve to continue or /skip to dismiss. Nothing fills until you answer.

Do I need to keep Telegram open?

No. Rhino runs continuously. Open Telegram when you want status, approvals, or to hit the kill switch.

Is this financial advice?

No. Rhino executes the rules you configure. Trading memecoins and stock tokens can result in total loss of funded capital.
